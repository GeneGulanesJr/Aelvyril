# Token Usage Field Reference

**Document version:** 1.0  
**Last updated:** 2026-04-23  
**Schema version:** 2

---

## TokenUsageEvent (One per LLM call)

| Field | Type | Example | Description |
|-------|------|---------|-------------|
| `schema_version` | `u32` | `2` | Schema version for migration support. Consumers must check this before parsing. |
| `event_id` | `String` | `uuid` | Unique identifier for deduplication. Events with the same `event_id` are idempotently upserted. |
| `timestamp` | `DateTime<Utc>` | `2026-04-23T14:30:00Z` | Server-side timestamp (rounded to minute for privacy). |
| `session_id` | `String` | `sess_7a3f...` | Opaque, non-sequential session identifier (UUID v4). |
| `tenant_id` | `String` | `"default"` | Tenant/org identifier. Single-tenant deployments use `"default"`. |
| `tool_name` | `String` | `"chat_completions"` | Which tool/feature triggered the LLM call. See `ToolName` enum for valid values. |
| `model_id` | `String` | `"gpt-4o"` | Model identifier as reported by the provider. |
| `retry_attempt` | `u32` | `0` | `0` = first attempt, `1+` = retry. Prevents double-counting on retry. |
| `tokens_in_system` | `u64` | `1200` | System prompt tokens (fixed overhead, not user-controllable). |
| `tokens_in_user` | `u64` | `8500` | User/context tokens (what the user/application sent). |
| `tokens_in_cached` | `u64` | `3400` | Cached prompt tokens (discounted rate). When `tokens_in_cache_write` is present, this is cache-read only. |
| `tokens_in_cache_write` | `u64` | `800` | Cache-write tokens (typically 25% MORE expensive than fresh input). v2 field. |
| `tokens_out` | `u64` | `2100` | Completion tokens generated by the model. |
| `tokens_truncated` | `u64` | `150` | Tokens discarded due to context window overflow. High values indicate prompt size issues. |
| `token_count_source` | `enum` | `"api_reported"` | How counts were obtained: `api_reported` (most accurate), `estimated` (local calculation), `unavailable` (not reported). |
| `was_streamed` | `bool` | `true` | Was this a streaming response? |
| `was_partial` | `bool` | `false` | Did the stream disconnect before completion? Partial streams have lower `tokens_out`. |
| `duration_ms` | `u64` | `1340` | Wall-clock time from request sent to response complete (ms). For streams: time to last chunk. |
| `actual_cost_cents` | `Option<u64>` | `Some(41)` | Provider-reported cost in integer cents. `None` when provider doesn't report cost. **Preferred over estimate.** v2 field. |
| `cost_estimate_cents` | `u64` | `42` | Cost calculated from token counts × pricing table. Fallback when `actual_cost_cents` is `None`. |
| `pricing_as_of` | `String` | `"2025-01-15"` | ISO date string indicating which pricing table was used. Pricing changes over time; this freezes the rate. |
| `cost_unavailable` | `bool` | `false` | `true` if pricing data was missing for this model. Shows "cost unavailable" in UI instead of `$0.00`. |
| `success` | `bool` | `true` | Did the call complete normally? `false` for errors, timeouts, partial streams. |

## Type Definitions

### TokenCountSource

- `ApiReported` — API returned `usage` data (most accurate)
- `Estimated` — Local estimation used (API didn't report)
- `Unavailable` — No counts available (null in output, not 0)

### SessionStatus

- `Active` — Session has recent activity
- `Closed` — Session explicitly closed
- `Orphaned` — Session auto-closed due to inactivity timeout

### ToolName (enum)

- `ChatCompletions` — `"chat_completions"`
- `Passthrough` — `"passthrough"`
- `OrchestratorPlan` — `"orchestrator_plan"`
- `OrchestratorExecute` — `"orchestrator_execute"`

New values must be added to both the enum and this document before use.
