# Phase 12: CLI — `aelvyril chat` Command

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the `aelvyril chat` CLI command — a readline loop that connects via WebSocket to the Orchestrator's Supervisor agent. For terminal-heavy users who don't want a browser. Same Supervisor, same chat as the web UI.

**Architecture:** A separate entry point in the same package. Uses the `ws` library to connect to `ws://localhost:PORT/ws`. Readline for user input. Displays agent responses and status updates in the terminal. Supports `--session <id>` flag to connect to a specific session.

**Tech Stack:** Node.js readline, ws (WebSocket client)

**Spec reference:** `docs/superpowers/specs/2026-05-05-cloud-platform-design.md` §3.4 (User interface)

**Depends on:** Phase 1-4 (Supervisor)
